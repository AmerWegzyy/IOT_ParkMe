# ParkMe — API & Database Structure

ParkMe is powered by a Firestore NoSQL database and a FastAPI REST backend. 

---

## 1. Firestore Database Schema

The database consists of 4 core collections:

### A. `parking_spots`
Represents the physical parking bays and maps hardware to logic.
* **Document ID:** Logical spot name (e.g., `"A1"`)
* `category` (string): `"student"`, `"lecturer"`, `"staff"`, `"special-needs-driver"`
* `sensor_mac` (string): The MAC address of the ultrasonic node
* `camera_mac` (string): The MAC address of the camera node
* `is_occupied` (boolean)
* `battery_level` (float)
* `last_seen` (timestamp)

### B. `users`
Represents the humans using the app.
* **Document ID:** User's email (e.g., `"student@technion.ac.il"`)
* `role` (string): Maps to spot categories, plus `"admin"`
* `name` (string)
* `points` (number)

### C. `vehicles`
Maps license plates to human users.
* **Document ID:** The license plate string (e.g., `"1234567"`)
* `user_id` (string): The email of the owner.

### D. `parking_logs`
An append-only ledger of every parking event (for analytics and violation tracking).
* **Document ID:** Auto-generated by Firestore
* `spot_id` (string): Logical spot (e.g., `"A1"`)
* `license_plate` (string): The plate, `"UNIDENTIFIED"`, or `"ABORTED"`
* `entry_time` (timestamp)
* `exit_time` (timestamp | null): Null means the car is currently parked there.
* `is_violation` (boolean)

*(Note: Querying `parking_logs` requires a Composite Index on `spot_id` + `exit_time` + `entry_time`)*

## 2. Core API Endpoints

All endpoints are hosted under `/api/v1`.

### Hardware Endpoints
* `POST /sensors/heartbeat`: Accepts JSON (`mac_address`, `is_occupied`, `battery_level`). Maps to the `sensor_mac` to update occupancy or spawn ghost logs.
* `POST /sensors/park`: Accepts `multipart/form-data` with `camera_mac` and `file`. Performs Cloud Vision OCR, maps to `camera_mac`, resolves violations, and opens the gate.
* `POST /telemetry/bulk`: Used by the Sensor Node to dump NVS-cached offline telemetry after network restoration.

### Client Endpoints
* `GET /users/me`: Requires JWT Auth header. Validates Firebase Token and returns user role.
* `GET /spots`: Returns a real-time snapshot of all parking spots. Strips plates from unauthorized users.
* `GET /stream`: An open TCP socket for Server-Sent Events (SSE). Streams `spot_update` and `log_event` payloads directly to the browser. Client disconnects are safely handled in a `finally` block to prevent queue leaks.
* `GET /logs`: (Admin Only). Returns a ledger of anomalies (violations, unidentified ghosts, aborted sessions). Legitimate, authorized parking events are intentionally skipped.
* `PUT /sensors/resolve`: (Admin Only). Used to manually overwrite an `"UNIDENTIFIED"` ghost log with a `"RESOLVED"` status.
